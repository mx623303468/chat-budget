# 聊天式记账 PWA 产品说明书（精简版）

## 一、产品概述

### 1. 产品定位
一款基于 **聊天输入交互** 的轻量记账应用，核心特点：
- 输入即记录（类似发消息）
- 自动滚动预算（每日限额累计）
- 极低学习成本
- 支持离线使用（PWA）

---

### 2. 核心目标
- 让用户 **1~2 秒完成一笔记账**
- 无需理解复杂财务结构
- 不打断、不跳转、不思考

---

### 3. 核心原则
1. **输入优先**：体验核心在输入效率
2. **强约束 + 强容错**：规则简单，但用户几乎不会输错
3. **不存派生数据**：所有余额通过计算得出
4. **默认正确**：用户不填，系统自动补

---

## 二、核心功能

---

### 1. 聊天式记账（核心功能）

#### 输入规则（对用户）
金额 + 空格 + 说明
示例：10 早餐

---

#### 实际支持（容错）
| 用户输入 | 系统处理 |
|----------|----------|
| `早餐 10` | 自动转为 `10 早餐` |
| `10早餐` | 自动补空格 |
| `１０ 早餐` | 转半角 |
| `10` | 自动补为 `10 未命名` |

---

#### 数据结构
```ts
type Transaction = {
  id: string
  amount: number
  note: string
  date: string
  createdAt: number
}
```

### 2. 滚动预算系统
#### 核心规则
今日余额 = 昨日余额 + 每日限额 - 今日支出

#### 推荐计算方式（最终实现）
余额 = 总预算 - 总支出

balance = totalDays * dailyLimit - totalSpend

特点
- 自动跨天累计
- 支持负数（超支）
- 不依赖每日计算

### 3. 核心 UI

#### 展示内容：

- 今日余额（大数字）
- 今日支出
- 状态颜色：
    - 正数 → 绿色
    - 0 → 黄色
    - 负数 → 红色

### 4. 聊天记录列表

#### 展示形式

[我] 10 早餐
[我] 20 午饭

#### 交互
- 左滑删除
- 长按编辑
- 自动滚动到底部交互
- 左滑删除
- 长按编辑

### 5. 设置
- 每日限额（必填）
- 起始日期（用于预算计算）

## 三、输入系统设计（核心模块）
### 1. 输入组件行为
- Enter 提交
- 提交后清空
- 保持焦点
- 不弹窗、不跳转

### 2. 实时提示（非阻断）
#### 示例
```
输入：
10
提示：
请输入说明（空格后）
```

```
输入：
早餐
提示：
请先输入金额，例如：10 早餐
```

### 3. 输入解析逻辑
核心规则
```ts
/^(\d+(\.\d+)?)\s+(.+)$/
```

### 4. 输入纠错（必须实现）

```ts
function normalize(input: string) {
  return input
    .replace(/[０-９]/g, s =>
      String.fromCharCode(s.charCodeAt(0) - 65248)
    )
    .replace(/^(\d+)([^\s\d]+)/, '$1 $2')
    .replace(/^([^\d]+)(\d+)/, '$2 $1')
    .replace(/\s+/g, ' ')
}
```

## 六、边界情况（必须处理）
1. 跨天未打开
- 使用累计计算，不依赖每日触发

2. 修改历史记录
- 自动重新计算余额

3. 删除记录
- 所有后续余额更新

4. 限额修改
- 需要选择今日立即生效，还是从明日开始生效
```ts
type LimitHistory = {
  date: string
  limit: number
}
```

5. 输入异常
| 问题    | 处理  |
| ----- | --- |
| 全角数字  | 转换  |
| 多空格   | 压缩  |
| 无说明   | 自动补 |
| 非数字开头 | 提示  |

6. 精度问题
- 输入的单位为元，例如  1.25 代表 一块两角五分
- 使用“分”为单位存储金额（整数）

## 一句话总结

让用户感觉不是在“记账”，而是在“发一条消息”
